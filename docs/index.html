How it works

Listen, clarify, translate, validate.

Four steps between your request and the model. The protocol stays the same whether you install the skill or run the MCP server.

1

Listen

Extract true intent — including unspoken constraints — from the user's request. Audience, purpose, form, hidden requirements.

2

Clarify

Ask one or two pointed questions when an answer would change the output by 50%+. Never interrogate.

3

Translate

Convert intent into the prompt shape the model performs best on. Lead with what serves the form.

4

Validate

Grade output against intent and re-run on miss. Self-validation in the skill, separate validator pass in the MCP server.
